# Data

This repository **does not commit** raw single-cell data, model checkpoints, embeddings, large
intermediate matrices, or BTLA source/derived data. It commits only: (a) small provenance and
coverage tables, (b) **manifests** (`data/manifests/`) listing every input with a checksum,
expected dimensions, source and access status, and (c) the notebooks that regenerate all outputs.

## ⚠️ Publication gate (BTLA data)

The **BTLA transcriptomic and multi-omic data are access-restricted and their publication
permission is unresolved.** Until clearance is confirmed in writing, do **not** commit or push:

* the BTLA differential-expression panel with effect sizes (`btla_4h_deg_gene_level_table.csv`);
* BTLA multi-omics evidence tables (`btla_candidate_multiomics_*.csv`);
* Paperclip BTLA literature tables;
* any figure or table that exposes BTLA effect sizes or per-target confirmed-gene lists.

These are listed in `.gitignore`. Running the notebooks regenerates them **locally**; they remain
untracked. When permission is granted, remove the corresponding `.gitignore` lines and re-commit.

## How to obtain / regenerate

1. **CD4 Perturb-seq**: download from the CZI genome-scale T-cell Perturb-seq dataset
   (https://virtualcellmodels.cziscience.com/dataset/genome-scale-tcell-perturb-seq) and place the
   DE-stats AnnData at the path given in `config/benchmark_config.yaml`. Verify with the checksum in
   `manifests/inputs_manifest.csv`.
2. **GREmLN checkpoint**: obtain `model.ckpt` from the GREmLN model page
   (https://virtualcellmodels.cziscience.com/model/gremln) into `third_party/GREmLN/checkpoints/`.
3. **BTLA data**: access-restricted. Contact the data owners; do not redistribute.
4. **Regenerate outputs**: run the notebooks in order (see top-level `README.md`). Notebook 01
   builds the GENIE3 masked graph; notebook 02 runs GREmLN inference (GPU); notebook 03 produces
   the benchmark tables and figures.

## `derived/`

Small, publication-permitted derived tables only (populated after data clearance). Currently empty
except a `.gitkeep`; the notebooks write regenerated outputs to `results/` (mostly gitignored until
clearance).
